认识Spring Security

本文主要讨论Spring Security的实际应用，即实现认证与授权（RBAC）功能。

Spring Security、OAuth2与JWT的关系

Spring Security 是一个在 Spring 框架上构建的用于处理身份验证和授权的框架。它提供了一套强大的安全性功能，可以帮助开发者保护他们的应用程序免受未经授权的访问和攻击。Spring Security 支持多种身份验证方法，包括基于用户名和密码的身份验证、LDAP、OAuth2 等。它还允许您定义细粒度的访问控制策略，以确保只有授权的用户能够执行特定的操作。

OAuth2 是一种开放标准协议，用于授权第三方应用程序访问用户帐户资源，而不需要共享用户的凭据。它允许用户授权应用程序访问他们的资源，例如照片、联系人列表或社交媒体帐户，而无需将用户名和密码分享给第三方应用程序。OAuth2 定义了不同的角色，包括资源所有者（用户）、客户端（第三方应用程序）和授权服务器（负责授权的服务器）。OAuth2 协议规定了如何进行身份验证和授权流程。

JWT（JSON Web Token） 是一种用于在不同系统之间安全传输信息的开放标准（RFC 7519）。它是一种轻量级、自包含的令牌，通常用于在身份验证和授权过程中传递信息。JWT 由三部分组成：头部（Header）、负载（Payload）和签名（Signature）。头部包含描述令牌类型和签名算法的信息，负载包含要传输的数据，签名用于验证令牌的完整性。JWT 可以被用于存储用户信息、授权信息以及其他任何需要安全传输的数据。

这些技术之间的关系如下：

  • Spring Security 和 OAuth2： Spring Security 可以与 OAuth2 集成，以实现安全的身份验证和授权流程。OAuth2 定义了一种授权机制，使得第三方应用程序可以通过用户的授权来访问受保护的资源。Spring Security 可以用于实施 OAuth2 中的授权服务器和资源服务器，以确保访问受保护资源的安全性和授权。
  • Spring Security 和 JWT： Spring Security 可以与 JWT 集成，以实现基于令牌的身份验证和授权。当用户成功登录后，Spring Security 可以生成一个包含用户信息和授权信息的 JWT，并将其返回给客户端。客户端在以后的请求中可以携带 JWT，Spring Security 可以验证令牌的有效性并允许或拒绝访问。
  • OAuth2 和 JWT： OAuth2 和 JWT 可以一起使用，以提供安全的授权流程。OAuth2 授权服务器可以生成包含访问令牌的 JWT，并将其返回给客户端。客户端可以使用访问令牌来访问受保护的资源，资源服务器可以验证 JWT 的有效性，并根据其中的信息决定是否授权访问。

总结

Security本身是一套完整的认证和授权解决方案，他是一条很长的过滤器链，使用oauth和JWT需要在过滤器链中定义oauth和JWT的具体实现。

前后端分离的系统，需要实现token的颁发和校验来进行用户认证。JWT是token的实现，做token的生成以及校验。

oauth2.0是规范，使用时是引入了oauth2.0的规范。系统需要给第三方系统做授权，就需要实现oauth。

原理

Spring Security的整体原理为：

  1. 当http请求进来时，使用severlet的Filter来拦截。
  2. 提取http请求中的认证信息，例如username和password，或者Token。
  3. 从数据库（或者其他地方，例如Redis）中查询用户注册时的信息，然后进行比对，相同则认证成功，反之失败。

一个请求过来Spring Security会按照下图的步骤处理：

  • Filter

拦截Http请求，获取用户名和秘密等认证信息

关键方法：

  • AuthenticationManager

从filter中获取认证信息，然后查找合适的AuthenticationProvider来发起认证流程

关键方法：

  • AuthenticationProvider

调用UserDetailsService来查询已经保存的用户信息并与从http请求中获取的认证信息比对。如果成功则返回，否则则抛出异常。

关键方法：

  • UserDetailsService

负责获取用户保存的认证信息，例如查询数据库。

关键方法：

这些组件都是抽象的，每个都可以有不同的实现，换句话说都是可以定制，特别灵活，所以就特别复杂。使用的一般是默认实现：

  • Filter： UsernamePasswordAuthenticationFilter
  • AuthenticationManager： ProviderManager
  • AuthenticationProvider： DaoAuthenticationProvider
  • UserDetailsService： InMemoryUserDetailsManager
